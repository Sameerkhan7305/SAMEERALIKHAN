API:
API stands for Application Programming Interface. It allows different software applications to communicate and exchange data with each other.

Client-Server Architecture:

Client-Server Architecture is a model where the client sends a request for data or service, and the server processes that request and sends back a response.

REST = Representational State Transfer

A REST API is a type of web API that allows a client and server to communicate using HTTP requests and resources identified by URLs.

Resource means the data/entity that the API manages.

An endpoint is a specific URL/path through which a client accesses an API resource or performs an operation.

JSON is a lightweight text format used to store and exchange data between different systems

The Request and Response Cycle is the process in which a client sends a request to a server, the server processes it, and sends a response back to the client.

HTTP Status Codes are three-digit numbers sent by the server to tell the client whether a request was successful, failed, redirected, or had some other result

1xx → Information

2xx → Success

3xx → Redirection

4xx → Client error (Client side request problem)

5xx → Server side error



200 → OK / Success

201 → New resource created

204 → Success but no response body

400 → Bad request

401 → Authentication required/failed

403 → No permission

404 → Resource not found

500 → Server error

There are different ways in JavaScript and React to send HTTP requests to a server and receive API responses.

1. Fetch API
Fetch API is a built-in browser API used to make HTTP requests and handle responses using Promises.
2. Axios Library
Axios is a promise-based HTTP client library used to make API requests from JavaScript applications.

Async/Await is a modern JavaScript syntax used to handle asynchronous operations, such as API calls, in a more readable and synchronous-looking way.

The async keyword is used before a function to make it an asynchronous function. An async function always returns a Promise.
